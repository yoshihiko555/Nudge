# ADR-0008: 設定画面を別ウィンドウに分離する

## 決定

1. メイン画面のタブは「タスク」「習慣」のみとし、設定タブは廃止する
2. 設定画面は専用ウィンドウで開き、メニューバーの「設定」およびヘッダーの設定アイコンから起動する
3. 設定専用ウィンドウではタブ表示とポーリングを無効化し、設定UIのみ表示する
4. 同一UI資産を使い、`?mode=settings` で表示モードを切り替える

## 理由

- 設定項目が増えており、ポップオーバー内では情報量が過密になりやすい
- メイン画面はタスク/習慣の確認に集中させ、設定は広い専用画面で編集できるようにする
- 既存のUI資産を再利用し、実装コストを抑えつつ体験を分離できる

## 代替案

- 既存の設定タブを維持 → 画面密度が高くなり可読性が下がるため不採用
- シート表示（同一ウィンドウ内） → 操作が煩雑になり、サイズ制限の課題が残るため不採用

## 影響

- `cmd/nudge/main.go`: 設定専用ウィンドウの追加と起動処理
- `cmd/nudge/assets/index.html`: ヘッダーに設定アイコン追加
- `cmd/nudge/assets/styles.css`: ヘッダー操作系のスタイルと設定モードの表示制御
- `cmd/nudge/assets/app.js`: モード判定と設定ウィンドウ起動RPCの追加
