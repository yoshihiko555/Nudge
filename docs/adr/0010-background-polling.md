# ADR-0010: バックグラウンド常時ポーリングとキャッシュ参照

## 決定
アプリ起動中はバックグラウンドで全DBを定期ポーリングし、UI はキャッシュ結果を参照する。

## 理由
- ウィンドウ非表示時も最新情報を維持したい
- ローカル完結の要件を満たしつつ実装を大きく増やさない
- 手動更新は強制リフレッシュで即時反映できる

## 代替案
- Webhook + 常駐サーバ（ローカル完結に反する）
- UI 表示時のみ都度取得（非表示中の鮮度が落ちる）

## 影響
- Notion API 呼び出し回数が増加する
- 取得失敗時はキャッシュ維持のため、最終更新の表示に注意が必要
